LeetCode-1579-保证图可完全遍历

发表于 2021-01-27 | 分类于 LeetCode
字数统计: 2.2k | 阅读时长 ≈ 8

题目描述

Alice 和 Bob 共有一个无向图，其中包含 n 个节点和 3 种类型的边：

类型 1：只能由 Alice 遍历。
类型 2：只能由 Bob 遍历。
类型 3：Alice 和 Bob 都可以遍历。
给你一个数组 edges ，其中 edges[i] = [typei, ui, vi] 表示节点 ui 和 vi 之间存在类型为 typei 的双向边。请你在保证图仍能够被 Alice和 Bob 完全遍历的前提下，找出可以删除的最大边数。如果从任何节点开始，Alice 和 Bob 都可以到达所有其他节点，则认为图是可以完全遍历的。

返回可以删除的最大边数，如果 Alice 和 Bob 无法完全遍历图，则返回 -1 。

示例 1：

输入：n = 4, edges = [[3,1,2],[3,2,3],[1,1,3],[1,2,4],[1,1,2],[2,3,4]]
输出：2
解释：如果删除 [1,1,2] 和 [1,1,3] 这两条边，Alice 和 Bob 仍然可以完全遍历这个图。再删除任何其他的边都无法保证图可以完全遍历。所以可以删除的最大边数是 2 。
示例 2：

输入：n = 4, edges = [[3,1,2],[3,2,3],[1,1,4],[2,1,4]]
输出：0
解释：注意，删除任何一条边都会使 Alice 和 Bob 无法完全遍历这个图。
示例 3：

输入：n = 4, edges = [[3,2,3],[1,1,2],[2,3,4]]
输出：-1
解释：在当前图中，Alice 无法从其他节点到达节点 4 。类似地，Bob 也不能达到节点 1 。因此，图无法完全遍历。

提示：

1 <= n <= 10^5
1 <= edges.length <= min(10^5, 3 n (n-1) / 2)
edges[i].length == 3
1 <= edges[i][0] <= 3
1 <= edges[i][1] < edges[i][2] <= n
所有元组 (typei, ui, vi) 互不相同

解题思路

思路理顺

读完题目很明显第一反应，找连通分量，而且还是分别找。分别对Alice和Bob寻找连通分量，看两人是否都能连接所有点。那么第一思路就是：并查集。并且是两个并查集，分别维护。

题目里的边分为三种，公共边、Alice独占边（简称A边）和Bob独占边（简称B边）。

题目要求删去最多的边，那么很明显，对于两人来说，任意两个节点之间使用公共边可以节约两条独占边（如果存在的话），而使用独占边则最少要两条。

因此，公共边的优先级是最高的。

我们遍历每一条公共边，对于其连接的的两个节点：

如果这两个节点在同一个连通分量中，那么这条边是可以去掉的。

如果这两个节点不在同一个连通分量中，我们就可以（并且一定）添加这条公共边，然后归并这两个节点所在的连通分量。

至此，我们完成了对公共边的添加，此时的图对于Alice和Bob来说是一样的，因为不存在任何独占边。

然后我们对A边和B边进行添加，因为两种边并不互通，所以当Alice不需要某A边时，该A边可以直接去掉，B边同理。

最后，完成所有归并以后，我们还需要满足Alice和Bob都可以完全遍历的条件，因此检查两个并查集最后是否都仅剩一个连通分量。

最后按题目要求给出结果即可。
